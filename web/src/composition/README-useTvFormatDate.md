ю# Форматирование дат в приложении

## Обзор

Для форматирования дат в приложении используется composable `useTvFormatDate`, который предоставляет гибкие возможности отображения дат в различных форматах с поддержкой локализации.

## Использование

```typescript
import { useTvFormatDate } from '@/composition/useTvFormatDate';

// Базовое использование
const formattedDate = useTvFormatDate(date);

// С опциями
const formattedDate = useTvFormatDate(date, {
    format: 'relative',
    showDayName: true,
    locale: 'ru'
});
```

## Опции форматирования

### format
- `'relative'` (по умолчанию) - относительные даты (Сегодня, Завтра, Через 2 дня)
- `'short'` - краткий формат (15.12 или 15.12.2023)
- `'full'` - полный формат (15 декабря 2023)

### showDayName
- `true` - добавляет день недели в скобках
- `false` (по умолчанию) - без дня недели

### showTime
- `true` - добавляет время (только для полного формата)
- `false` (по умолчанию) - без времени

### locale
- `'ru'` - русская локализация
- `'en'` - английская локализация
- Если не указано, используется текущая локаль приложения

## Примеры использования

### 1. Список задач (относительный формат)
```typescript
const taskDate = useTvFormatDate(task.startDate, {
    format: 'relative',
    showDayName: true
});
// Результат: "Завтра", "Через 2 дня", "15.12 (Пт)"
```

### 2. Таблица (краткий формат)
```typescript
const tableDate = useTvFormatDate(task.startDate, {
    format: 'short'
});
// Результат: "15.12", "15.12.2023"
```

### 3. Детальная страница (полный формат)
```typescript
const detailDate = useTvFormatDate(task.startDate, {
    format: 'full',
    showTime: true
});
// Результат: "15 декабря 2023 14:30"
```

## Рекомендации по UX

### 1. Контекст использования
- **Списки и дашборды**: используйте относительный формат
- **Таблицы**: используйте краткий формат
- **Детальные страницы**: используйте полный формат

### 2. Относительные даты
- Показывайте "Сегодня", "Завтра", "Вчера" для близких дат
- Для дат в пределах недели используйте "Через X дней" или "X дней назад"
- Для более дальних дат используйте абсолютный формат

### 3. День недели
- Добавляйте день недели для дат, которые не являются сегодня/завтра
- Это помогает пользователям лучше ориентироваться в календаре

### 4. Локализация
- Всегда учитывайте локаль пользователя
- Используйте правильные окончания для русского языка
- Адаптируйте форматы под культурные особенности

## Лучшие практики

1. **Консистентность**: используйте одинаковые форматы в похожих контекстах
2. **Читаемость**: избегайте слишком длинных форматов в компактных интерфейсах
3. **Доступность**: убедитесь, что даты читаются скринридерами
4. **Производительность**: используйте computed для реактивных дат
5. **Тестирование**: тестируйте с различными датами и локалями
